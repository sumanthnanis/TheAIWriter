# Abstract

The development of an e-commerce mobile application for Self Help Groups (SHGs) trained in product-making presents a unique solution to facilitate the sale of their products. This paper documents the process and technology stack employed in the creation of the app, highlighting the use of Flutter, Dart, Node.js, and MySQL. The choice of these technologies was driven by their ability to provide a seamless cross-platform experience, efficient coding, and scalable backend operations. The use of Flutter's hot reload feature, along with its single codebase approach, streamlined the development process, enabling the delivery of a polished user experience. Additionally, the versatility and ease of learning Dart language, coupled with the scalability and efficiency of Node.js, contributed to the robustness of the application. The incorporation of a structured MySQL database further enhanced the application's data management capabilities, offering visualizations through tables. This paper aims to showcase the benefits of the chosen technology stack in addressing the specific needs of SHGs selling their products through an e-commerce platform.Introduction

The methodology adopted for the development of the e-commerce mobile application for self-help groups (SHGs) involved the use of specific technologies and frameworks to ensure an efficient and scalable platform. The chosen technologies were selected based on their ability to provide a seamless cross-platform experience, ease of development, scalability, and database management. 

## Technology Stack

### 1. Flutter
Flutter was the primary choice for developing the mobile application due to its cross-platform compatibility and single codebase feature. This allowed for the efficient creation of a stunning user interface and experience using a unified codebase. The implementation of Flutter significantly reduced the development time and provided a polished user experience. Additionally, the hot reload feature of Flutter facilitated real-time changes to the application, accelerating the development process.

### 2. Dart
Dart, as the language powering Flutter, played a pivotal role in writing clean and maintainable code for both the frontend and backend components of the application. Its versatility and ease of learning made it an ideal fit for the project's requirements, ensuring efficient development and easy maintenance.

### 3. Node.js
Node.js was employed for the backend infrastructure of the e-commerce application due to its scalability and event-driven architecture. The event-driven nature of Node.js made it suitable for handling I/O-bound operations, such as serving HTTP requests and interacting with databases, thereby ensuring the efficiency and responsiveness of the application. Additionally, the vast ecosystem of npm packages available with Node.js contributed to the extensibility and robustness of the backend infrastructure.

### 4. MySQL
The choice of MySQL as the structured database management system provided a reliable and cost-effective solution for data storage and management. The use of MySQL allowed for efficient data retrieval and storage, while also enabling easy visualizations through the use of tables, ensuring intuitive and seamless data management for the e-commerce platform.

## Conclusion
The methodology encompassing the use of Flutter, Dart, Node.js, and MySQL provided a robust foundation for the development of the e-commerce mobile application for SHGs. **Dart Code Sample**

```dart
import 'package:flutter/material.dart';
import 'package:http/http.dart' as http;

class Product {
  final String id;
  final String name;
  final double price;

  Product({this.id, this.name, this.price});

  factory Product.fromJson(Map<String, dynamic> json) {
    return Product(
      id: json['id'],
      name: json['name'],
      price: json['price'],
    );
  }
}

Future<Product> fetchProduct() async {
  final response = await http.get('https://api.example.com/products/1234');
  
  if (response.statusCode == 200) {
    return Product.fromJson(json.decode(response.body));
  } else {
    throw Exception('Failed to load product');
  }
}
```

**Node.js Code Sample (Backend Route)**

```javascript
const express = require('express');
const app = express();
const port = 3000;

app.get('/products/:id', (req, res) => {
  const productId = req.params.id;
  // Fetch product details from the database and send response
  res.json({ id: productId, name: 'Product ABC', price: 49.99 });
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});
```
